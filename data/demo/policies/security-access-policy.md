# Atlas Security Access Policy

This is fictional demo data created for the MiniRAG Assistant portfolio project.

Access must follow the principle of least privilege. Users receive only the
permissions required for their current responsibilities, and every new access
request requires manager approval before access is granted.

Application owners must review access at least every 90 days. When a person
leaves the organization, their application access must be disabled within 24
hours after departure.

Authentication secrets must be protected. Refresh tokens must not appear in
application logs, audit messages, analytics events, or support diagnostics.
